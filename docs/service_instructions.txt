Windows service / background runner instructions

Options to run the Video.AI worker as a persistent background process on Windows:

1) NSSM (recommended for simple services)
 - Download NSSM: https://nssm.cc/download
 - Install service:
   nssm install videoai-worker "C:\Video.AI\venv\Scripts\python.exe" "C:\Video.AI\-m" "jobs.run_worker"
   (Alternatively set Application to C:\Windows\System32\cmd.exe and Arguments to "/c start "" C:\Video.AI\run_worker.bat"")
 - Configure stdout/stderr file paths in NSSM GUI to C:\Video.AI\logs\worker.log
 - Start service: nssm start videoai-worker

2) Scheduled Task at logon (no admin required)
 - Create a scheduled task that runs at user logon and starts run_worker.bat hidden.
 - Example:
   schtasks /Create /TN "VideoAI Worker" /TR "C:\Video.AI\run_worker.bat" /SC ONLOGON /RL HIGHEST /F
 - To run immediately:
   schtasks /Run /TN "VideoAI Worker"

3) Windows Service with sc.exe (advanced)
 - Running arbitrary Python scripts as a native service is more involved. Use a service wrapper like NSSM or create a Python service with pywin32.

Notes:
 - Ensure the venv is activated by the run_worker.bat (it calls venv\Scripts\activate.bat).
 - Logs directory: create C:\Video.AI\logs and ensure the service account has write permission.
 - For troubleshooting, run run_worker.bat manually to validate behavior before installing as a service.

PowerShell convenience script:
 - scripts\run_worker_service.ps1 starts the worker detached. It is not a full service; prefer NSSM or scheduled task for production.
