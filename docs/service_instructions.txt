Windows service / background runner instructions

The default background worker is the Python queue worker started by
run_worker.bat. The optional Rust worker sidecar is documented in README.md.

Option 1: NSSM

1. Download NSSM from https://nssm.cc/download.
2. Install the service:

   nssm install videoai-worker "C:\Video.AI\run_worker.bat"

3. Configure stdout/stderr paths to:

   C:\Video.AI\logs\worker.log

4. Start the service:

   nssm start videoai-worker

Option 2: Scheduled Task at logon

Create a scheduled task that runs at user logon:

   schtasks /Create /TN "VideoAI Worker" /TR "C:\Video.AI\run_worker.bat" /SC ONLOGON /RL HIGHEST /F

Run immediately:

   schtasks /Run /TN "VideoAI Worker"

Notes

- `run_worker.bat` activates the project venv before starting the worker.
- Ensure `C:\Video.AI\logs` exists and is writable by the service account.
- Before installing a service, run `run_worker.bat` manually once to validate the environment.
- `scripts\run_worker_service.ps1` starts the worker detached, but it is not a real Windows service.
