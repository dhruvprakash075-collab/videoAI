**Score: 8/10** — Coherent GitHub-dark palette. Two issues:

### Design System Issues

**Issue DS-1 (HIGH): `border: round` on Header and Footer clips content**
- Root cause: Header and Footer are single-row chrome widgets. A `round` border consumes the row.
- Fix (Task 1): Remove `border` from Header and Footer entirely.

**Issue DS-2 (MEDIUM): No CSS custom properties — tokens are hardcoded hex everywhere**
- The same `#0d1117` appears 3 times, `#30363d` appears 5 times.
- **Textual 8.2.7 does NOT support `var()` CSS custom properties** — tested and confirmed.
  The parser rejects `--token: value` declarations with a parse error.
- Fix: Keep hex values in CSS; define the palette as Python class constants instead
  (already done as `_STYLE_AGENT`, `_STYLE_USER`, etc.). Document the token mapping
  in a comment block above the CSS string so future maintainers can update consistently.

**Issue DS-3 (LOW): `.system`, `.agent`, `.user`, `.error` CSS classes are defined but never used**
- These were the original (broken) style names. They were fixed to use hex in Python.
- The CSS class definitions are now dead code.
- Fix: Remove the 4 dead CSS class rules.

**Issue DS-4 (LOW): `#status_bar` is a legacy widget that will be replaced**
- The new design uses a badge + meta line. The `#status_bar` CSS rule will be dead after Task 6.
- Note for builder: remove it during Task 6.

## 5. Frontend Design Direction (TUI-adapted)

**Purpose:** Operator control panel for a long-running GPU video pipeline.
**Audience:** Single operator, repeated daily use, needs to scan status at a glance.
**Tone:** Dense, quiet, industrial — like a terminal monitoring tool, not a dashboard.
**Memorable detail:** The spinner character cycling through braille dots (`⠋⠙⠹⠸⠼⠴⠦⠧⠇⠏`)
is already the right call — it communicates "working" without being distracting.

### Design direction recommendations for the rewrite

1. **Visual hierarchy:** Status badge (largest, top) → meta line (medium) → progress bar
   (thin, accent color) → log (body text, muted) → composer (focused, accent border).
   The operator's eye should land on status first, then progress, then log.

2. **Information density:** The current layout is too sparse. The new design should pack
   more into the status area: `⠋ RUNNING • topic • ⏱ 04:12 • seg 3/12 • ETC ~8m`.
   This is a tool for repeated use — density is a feature, not a problem.

3. **Color discipline:** Use the 4-color semantic palette consistently:
   - Blue (`#58a6ff`) = agent/system activity (Director, Writer, pipeline events)
   - Green (`#7ee787`) = operator input + success
   - Amber (`#e3b341`) = warnings, VRAM pressure, paused state
   - Red (`#f85149`) = errors, OOM events, fatal
   - Grey (`#7d8590`) = system/background noise

4. **The log is the product** — it's what the operator reads during a 2-hour run.
   Give it maximum vertical space. The status panel should be compact (3 rows max).

5. **Anti-patterns to avoid in the rewrite:**
   - Don't add decorative borders to chrome (Header/Footer) — already the bug we fixed.
   - Don't use color alone for status — always pair with an icon (already in the spec).
   - Don't animate the spinner with CSS — Textual's poll loop handles it naturally.

## 6. Agent Introspection — Known Failure Patterns for the Builder

When implementing the TUI, watch for these failure patterns:

### Pattern 1: "I fixed it but the PTY still shows the old layout"
**Cause:** Running in the embedded editor terminal, not a real PTY.
**Recovery:** Always test in Windows Terminal or PowerShell. Use `launch_tui.ps1`.

### Pattern 2: "The progress bar shows 0% and never advances"
**Cause:** `UIState.segment_total` is 0 (planning state). The pipeline hasn't called
`set_progress(total=n_segs)` yet, or the hook wasn't added to `pipeline_long.py`.
**Recovery:** Check Task 4 is complete. Verify `_director_set_abort(False)` is at the
top of `run_long_pipeline()` (Task 0 fix).

### Pattern 3: "The 2nd run shows no logs"
**Cause:** `log_index` not reset when `UIState.logs` is cleared.
**Recovery:** Add `if len(UIState.logs) < self.log_index: self.log_index = 0` at the
top of the log-draining block in `_poll()` (Task 8).

### Pattern 4: "Cancel works but the next run skips all segments"
**Cause:** `_director_abort` flag not reset. Task 0 fix adds `_director_set_abort(False)`
to `run_long_pipeline()`. Task 8 adds it to `on_input_submitted()` as belt-and-suspenders.

### Pattern 5: "The pause prompt disappears before I can reply"
**Cause:** `_poll()` sets `UIState.active_question = None` to prevent repeated display,
but this races with the pipeline thread. Fix: use `self._last_question` comparison
instead of mutating UIState (Task 8).

### Pattern 6: "Tests pass individually but fail when run together"
**Cause:** UIState is a class with global state. Tests bleed between runs.
**Recovery:** Add the `autouse=True` conftest.py fixture that resets ALL UIState fields
before each test (Task 3.1).
